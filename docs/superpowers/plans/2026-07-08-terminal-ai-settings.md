# Terminal AI Settings Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build usable local-client UI support for terminal regex highlighting, selected terminal text quotes into AI chat, and light/dark theme switching.

**Architecture:** Keep the current React shell, but move pure settings logic into small library modules that are easy to test. App state owns theme, highlight rules, and AI quotes; terminal rendering consumes those states without changing SSH/raw terminal data.

**Tech Stack:** React, TypeScript, xterm.js, Vitest, Testing Library, Tailwind CSS.
